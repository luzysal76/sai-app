### 🔍 현재 상태 정밀 분석

#### ✅ 정상 동작
- **로컬 챗봇**: Groq(`llama-3.1-8b-instant`) → OpenAI → Claude → 스마트 규칙 4단계 체인
- **배포 챗봇**: `apiBase = ''` → 서버 시도 생략 → 로컬 스마트 폴백 자동 동작 (에러 없음)
- **16개 도구**: 전수 점검 완료, 콘솔 에러 0

#### ❌ 현재 이슈 3가지

**이슈 1: 캡처 분석 완전 불가 (로컬·배포 모두)**
```
functions/.env: ANTHROPIC_API_KEY=sk-ant-여기에-실제-키-입력
↓
server.js loadApiKey() → null 반환
↓
/api/capture → 503 에러 반환 (시뮬레이션 폴백 없음!)
```
- chatbot.js와 달리 capture는 에러를 그냥 503으로 반환
- 사용자가 캡처 분석 시도하면 에러 메시지만 노출됨

**이슈 2: 배포 환경 AI 챗봇 = 스마트 규칙만**
```
config.js: apiBase = window.SAI_API_BASE || (isLocal ? 'http://localhost:4321' : '')
↓
Firebase Hosting에서 window.SAI_API_BASE 미주입
↓
apiBase = '' → 서버 없음 → 로컬 스마트 폴백만 동작
```
- Render/Railway에 server.js 배포 후 window.SAI_API_BASE 주입하면 해결

**이슈 3: Groq 모델 품질 저하**
```
functions/.env: GROQ_MODEL=llama-3.1-8b-instant (8B 소형 모델)
server.js: process.env.GROQ_MODEL || 'llama-3.3-70b-versatile'
↓
.env 값이 우선 → 8B 모델 사용 중 (한국어 품질 낮음)
```
- `.env`에서 `GROQ_MODEL=llama-3.1-8b-instant` 삭제하면 기본값 70B로 업그레이드
